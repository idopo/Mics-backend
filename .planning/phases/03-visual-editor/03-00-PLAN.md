## Background

mics_task.py already has:
- `_COMPARE` dict with `==`/`!=`/`>=` etc. — used by if-actions
- `_build_condition_operand` — resolves {tracker/flag/param/hardware} or literal
- `_build_if_action` — uses `_COMPARE` and `_build_condition_operand`
- `_build_transition_lambda` — separate, uses its own OP_MAP with `eq`/`ne` strings, only reads `view` key

The goal: make transitions use the same rich condition format as if-actions.

Canonical v2 condition (unified):
```json
{ "left": {"view": "licker1"}, "op": ">=", "right": 1 }
```

Legacy transition condition (still supported):
```json
{ "view": "licker1", "op": "eq", "rhs": 1 }
```

Both formats work after this plan. New FDAs created by the GUI will use the unified format.

```python
    def _build_transition_lambda(self, cond: dict):
        """
        Build a zero-argument callable from a transition condition dict.

        Supports two formats:
          New unified: {"left": <operand>, "op": ">=", "right": <operand>}
            where operand is {view/tracker/flag/param/hardware: name} or a literal.
          Legacy:      {"view": "<key>", "op": "eq", "rhs": <value_or_ref>}
            where op is eq/ne/lt/le/gt/ge or ==/!=/>=/<=/>/<.

        The unified format is produced by the GUI. The legacy format is supported
        for backward compatibility with hand-authored v1/v2 JSON.
        """
        op_str = cond.get("op", "")
        if op_str not in _COMPARE:
            raise ValueError(
                f"_build_transition_lambda: unknown op '{op_str}'. "
                f"Allowed: {sorted(_COMPARE.keys())}"
            )
        compare = _COMPARE[op_str]

        # ── New unified format: {left, op, right} ────────────────────────────────────────
        if "left" in cond:
            left_fn  = self._build_condition_operand(cond["left"])
            right_fn = self._build_condition_operand(cond["right"])
            def _unified_check(
                _l=left_fn, _r=right_fn, _c=compare,
            ):
                return _c(_l(), _r())
            return _unified_check

        # ── Legacy format: {view, op, rhs} ───────────────────────────────────────────────
        view_key = cond["view"]
        rhs_spec = cond["rhs"]   # may be literal or {"param":...} / {"flag":...}
        def _legacy_check(
            _view_key=view_key,
            _op_fn=compare,
            _rhs_spec=rhs_spec,
        ):
            current = self.view.get_value(_view_key)
            rhs     = self._resolve_arg(_rhs_spec)
            return _op_fn(current, rhs)
        return _legacy_check
```
