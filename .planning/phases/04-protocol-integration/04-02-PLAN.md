<objective>
Swap the data source in the protocol builder and overrides modal from raw task types to task definitions backed by toolkit metadata.

ProtocolsCreate: the left palette now shows task definitions (from GET /api/task-definitions filtered to those with fda_json). Clicking one adds a step whose param inputs are populated from the linked toolkit's params_schema. The step is saved with task_definition_id so the orchestrator can inject fda_json at session start. Graduation config is unchanged.

OverridesModal: when a protocol step carries task_definition_id, resolve params_schema from the toolkit instead of from getLeafTasks. Legacy steps (task_definition_id=null) still use the existing getLeafTasks path so nothing breaks.

Visual/functional UX stays the same — layout, param inputs, graduation, run history tab are identical. Only the data source changes.
</objective>
