# Phase 11, Plan 2: Pi Dispatch Class Fix + Condition Builder HW Modules

**Requirements:** HW-17, HW-18
**Phase goal:** Backend-authored toolkits dispatch the correct task class to the Pi; condition builder shows hardware module names alongside semantic_hardware keys.

**Gate:** Requires Phase 11-01 complete — `available_locked_states` table and toolkit backend-authored creation must exist.

---

## Context

Backend-authored toolkits have no source file on the Pi. When a run starts, the orchestrator sends `task_type` to the Pi to identify which class to instantiate. Currently `task_type` comes from the protocol step, which doesn't know about backend-authored toolkits. This plan:

1. **11-02-A**: Stores `class_name` in `available_locked_states` (from HANDSHAKE), exposes a dispatch-class endpoint, and has the orchestrator use it when building the START payload.
2. **11-02-B**: Fixes the condition builder in TaskEditor to show hardware module names (from `toolkit.hardware_module_ids`) alongside `semantic_hardware` keys in the HW dropdown.

**Key constraint:** `_build_first_step_task()` and `_build_step_task()` both re-assert `task["task_type"] = step["task_type"]` at the end as override protection (lines 686, 745). The `task_type` override must happen in `start_run()` and `_advance_run_step()` **after** these methods return — never inside them.

**Key constraint:** `api/routers/toolkits.py` is already at 595 lines (hard limit 500). New dispatch endpoints go in `api/routers/toolkit_dispatch.py`.

```python
def upsert_locked_states(self, pilot_id: int, task_filename: str, state_names: list[str], class_name: str | None = None):
    payload = {"state_names": state_names}
    if class_name is not None:
        payload["class_name"] = class_name
    r = self._put(f"/api/locked-states/{pilot_id}/{task_filename}", payload)
    r.raise_for_status()
```

### Step 5 — New file: `api/routers/toolkit_dispatch.py`

Create this file. Import pattern from `api/routers/locked_states.py` (same auth/db setup).

Endpoint: `GET /api/toolkits/{toolkit_id}/dispatch-class`

```python
@router.get("/toolkits/{toolkit_id}/dispatch-class")
def get_dispatch_class(toolkit_id: int, _: dict = Depends(verify_token), db=Depends(get_sa_session)):
    row = db.execute(
        text("SELECT locked_state_source, is_backend_authored FROM task_toolkits WHERE id = :id"),
        {"id": toolkit_id}
    ).fetchone()
    if not row:
        raise HTTPException(status_code=404, detail="Toolkit not found")
    if not row.locked_state_source:
        return {"class_name": "mics_task", "is_backend_authored": row.is_backend_authored}
    cls_row = db.execute(
        text("SELECT class_name FROM available_locked_states WHERE task_filename = :fname LIMIT 1"),
        {"fname": row.locked_state_source}
    ).fetchone()
    class_name = cls_row.class_name if cls_row and cls_row.class_name else "mics_task"
    return {"class_name": class_name, "is_backend_authored": row.is_backend_authored}
```

Note: `task_toolkits` is a SQLAlchemy table — use `get_sa_session` (not SQLModel session).
