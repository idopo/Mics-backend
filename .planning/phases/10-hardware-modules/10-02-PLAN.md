## Implementation

### Step 1: Hardware Modules page

**File:** `web_ui/react-src/src/pages/hardware-modules/HardwareModules.tsx` (new)

Layout: two-panel split (list left, create/edit right). Use existing CSS classes: `container split`, `card`, `scroll-list`, `params-grid`, `button-primary`, `button-secondary`.

**Left panel ‚Äî module list:**
- Fetch `GET /api/hardware-modules` on mount
- Each row: `name` | lib filename chip | `class_name` chip | Edit / Delete buttons
- Clicking a row populates right panel for edit

**Right panel ‚Äî create/edit form:**
- Name input
- Hardware lib dropdown (fetch `GET /api/hardware-libs`)
- Class name dropdown ‚Äî populated from selected lib's `ast_metadata.classes[].name`
  - Fetch `GET /api/hardware-libs/{id}` on lib select to get ast_metadata
  - Populate dropdown from `ast_metadata.classes.map(c => c.name)`
- Description textarea (optional)
- Save / Cancel buttons ‚Üí `POST /api/hardware-modules` or `PUT /api/hardware-modules/{id}`
- On 422 from API: show inline error (class not found in AST)

**Nav entry:** Add to `Layout.tsx` `NAV_LINKS` array with a chip/circuit SVG icon. Use `to: '/hardware-modules-ui'` ‚Äî no `/react/` prefix, BrowserRouter `basename="/react"` handles it:
```typescript
{ to: '/hardware-modules-ui', label: 'Hardware Modules', exact: false, icon: <CircuitIcon /> }
```

**Route:** Register in `App.tsx`. Path must NOT include `/react/` prefix (matches existing patterns like `"researchers-ui"`, `"hardware-libs-ui"`):
```typescript
<Route path="hardware-modules-ui" element={<HardwareModules />} />
```

### Step 2: Pilot Hardware Config editor

**File:** `web_ui/react-src/src/pages/hardware-modules/PilotHardwareConfig.tsx` (new)

Route: `pilots/:pilotId/hardware-config` (registered in `App.tsx` without `/react/` prefix)

Layout: table of all hardware modules with per-module config fields.

**Data loading:**
1. `GET /api/hardware-modules` ‚Äî full module list
2. `GET /api/pilots/{pilotId}/hardware-config` ‚Äî existing config rows for this pilot
3. For each module, `GET /api/hardware-modules/{id}/methods` to get `__init__` args ‚Üí derive config field names + types

**Table structure:**
- Column 1: module name + class chip
- Column 2+: config fields ‚Äî one input per `__init__` arg (excluding `self`)
  - `int` annotation ‚Üí `<input type="number" step="1" />`
  - `float` annotation ‚Üí `<input type="number" step="0.01" />`
  - `bool` annotation ‚Üí `<input type="checkbox" />`
  - anything else ‚Üí `<input type="text" />`
  - Show arg name + annotation as label; pre-fill `default` if present
- Edit button ‚Üí makes row inputs active
- Save button ‚Üí `PUT /api/pilots/{pilotId}/hardware-config/{moduleId}` with `{config: {<arg_name>: <value>, ...}}`

Pre-populate from existing config rows where present.

**Link from Pilot card:** Pilot cards in `/react/` (Index.tsx) may optionally link to this page ‚Äî not required in Phase 10.
