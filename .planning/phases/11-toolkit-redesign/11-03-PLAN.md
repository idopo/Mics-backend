# Phase 11, Plan 3: Dynamic HARDWARE Dispatch (Backend-Authored Instantiation)

**Requirements:** HW-19
**Phase goal:** Backend-authored toolkits run on the Pi without any source file — HARDWARE dict, FLAGS, PARAMS, and pin config are all sent from the backend in the START payload. Pi instantiates `mics_task` directly using injected data.

**Gate:** Requires Phase 11-02 complete — `toolkit_dispatch.py` router exists, `is_backend_authored` check in orchestrator is in place, toolkit_id is available in both `start_run()` and `_advance_run_step()`.

---

## Context

Phase 11-02 fixes *which class* the Pi instantiates. This plan completes the picture: for backend-authored toolkits, the Pi also needs the HARDWARE dict (with source code for each module), pin config, FLAGS, and PARAMS — none of which exist in a source file on the Pi.

**Backward compatibility:** `init_hardware()` and `init_flags()` are untouched. `self.HARDWARE`, `self.FLAGS`, `self.PARAMS`, and `prefs.HARDWARE` are populated before those methods run. Pi code that doesn't receive these keys runs unchanged.

**HARDWARE dict nesting mirrors legacy format** so `init_hardware()` works unchanged:
```
{"Modules": {"module_name": {"module_name": {"class_name": "...", "source_code": "..."}}}}
```
All modules go into implicit group `"Modules"`. No new group column needed.

**Key constraint:** `_build_*_task()` re-asserts reserved keys at the end (including `task_type`). All injection happens in `start_run()` and `_advance_run_step()` AFTER those methods return — never inside them.

**Key constraint:** `HardwareModule` has no SQLAlchemy relationship to `HardwareLib`. Use two explicit queries: fetch `HardwareLib` by `module.hardware_lib_id`, then fetch `HardwareLibVersion` by `lib.active_version_id`.

## Step 1 — Add dispatch-spec endpoint to `api/routers/toolkit_dispatch.py`

This file was created in Plan 11-02. Add a second endpoint to it.

`GET /api/toolkits/{toolkit_id}/dispatch-spec?pilot_id={pilot_id}`

```python
@router.get("/toolkits/{toolkit_id}/dispatch-spec")
def get_dispatch_spec(toolkit_id: int, pilot_id: int, _: dict = Depends(verify_token), db=Depends(get_sa_session)):
    toolkit = db.execute(
        text("SELECT id, hardware_module_ids, flags, params_schema, is_backend_authored FROM task_toolkits WHERE id = :id"),
        {"id": toolkit_id}
    ).fetchone()
    if not toolkit:
        raise HTTPException(status_code=404, detail="Toolkit not found")

    hardware = {}
    prefs_hardware = {}

    for module_id in (toolkit.hardware_module_ids or []):
        # Step 1: fetch module
        module = db.execute(
            text("SELECT id, name, class_name, hardware_lib_id FROM hardware_modules WHERE id = :id"),
            {"id": module_id}
        ).fetchone()
        if not module:
            continue

        # Step 2: fetch lib to get active_version_id
        lib = db.execute(
            text("SELECT active_version_id FROM hardware_libs WHERE id = :id"),
            {"id": module.hardware_lib_id}
        ).fetchone()
        if not lib or not lib.active_version_id:
            continue

        # Step 3: fetch version source code
        version = db.execute(
            text("SELECT source_code FROM hardware_lib_versions WHERE id = :id"),
            {"id": lib.active_version_id}
        ).fetchone()
        if not version:
            continue

        hardware.setdefault("Modules", {})[module.name] = {
            module.name: {
                "class_name": module.class_name,
                "source_code": version.source_code,
            }
        }

        # Step 4: fetch per-pilot pin config (may not exist)
        cfg = db.execute(
            text("SELECT config FROM pilot_hardware_configs WHERE pilot_id = :pid AND hardware_module_id = :mid"),
            {"pid": pilot_id, "mid": module_id}
        ).fetchone()
        if cfg:
            prefs_hardware.setdefault("Modules", {})[module.name] = cfg.config

    return {
        "hardware": hardware,
        "prefs_hardware": prefs_hardware,
        "flags": toolkit.flags or {},
        "params_schema": toolkit.params_schema or {},
        "is_backend_authored": toolkit.is_backend_authored,
    }
```

def _inject_backend_toolkit_spec(self, task: dict, toolkit_id: int, pilot_db_id: int) -> None:

```python
def _resolve_hardware_classes(self, hardware_dict: dict) -> dict:
    import importlib
    resolved = {}
    for group, group_entries in hardware_dict.items():
        resolved[group] = {}
        for hw_id, inner in group_entries.items():
            resolved[group][hw_id] = {}
            for sub_id, cls_spec in inner.items():
                if isinstance(cls_spec, dict):
                    # {"class_name": "...", "source_code": "..."} — exec to get the class
                    ns = {}
                    exec(cls_spec["source_code"], ns)  # noqa: S102
                    resolved[group][hw_id][sub_id] = ns[cls_spec["class_name"]]
                elif isinstance(cls_spec, str):
                    # "module.path.ClassName" — importlib
                    mod_path, cls_name = cls_spec.rsplit(".", 1)
                    mod = importlib.import_module(mod_path)
                    resolved[group][hw_id][sub_id] = getattr(mod, cls_name)
                else:
                    resolved[group][hw_id][sub_id] = cls_spec
    return resolved

def _merge_prefs_hardware(self, prefs_hw: dict) -> None:
    from autopilot import prefs as autopilot_prefs
    existing = autopilot_prefs.get("HARDWARE") or {}
    for group, entries in prefs_hw.items():
        existing.setdefault(group, {}).update(entries)
    autopilot_prefs.set("HARDWARE", existing)
```

## Test Vehicle: elastic_test.py

`pilot/plugins/elastic_test.py` is the recommended integration test vehicle for this plan.

- `elastic_test` inherits `HARDWARE` from `learning_cage` (class attribute — GPIO/I2C/Timers/Mixer).

- `FLAGS` and `PARAMS` injection can also be verified: `elastic_test`'s class-level dicts will be shadowed by the injected instance values.
