# Phase 11, Plan 5: Toolkit Edit — HW Modules, Flags, Params

**Phase goal:** Users can update hardware modules, flags, and params schema on an already-created
backend-authored toolkit without deleting and recreating it.

**Scope:** PATCH endpoint (backend) + Edit button + modal (UI). No impact detection — that is
Phase 12-02's responsibility. No changes to `locked_state_source` or `states` (structural fields
that would break linked task definitions in unpredictable ways).

---

## Implementation

### Step 1: Backend — PATCH endpoint

**File:** `api/routers/toolkits.py`

Add a Pydantic model alongside the existing `BackendToolkitCreate`:

```python
class BackendToolkitPatch(BaseModel):
    hardware_module_ids: List[int] | None = None
    flags: List[FlagDefinition] | None = None
    params_schema: List[ParamDefinition] | None = None
```

Add the endpoint:

```python
@router.patch("/toolkits/{toolkit_id}", status_code=200)
def patch_backend_toolkit(
    toolkit_id: int,
    payload: BackendToolkitPatch,
    _: dict = Depends(verify_token),
    db: Session = Depends(get_db),
):
```

Logic:
1. Fetch toolkit; 404 if not found or `is_backend_authored=False`
2. If `hardware_module_ids` provided:
   - Validate each id exists in `hardware_modules` table; 422 if any missing
   - Recompute `hw_hash = hashlib.md5(json.dumps(sorted(hardware_module_ids)).encode()).hexdigest()`
   - Update column
3. If `flags` provided: convert list → dict and update `flags` column (same transform as POST)
4. If `params_schema` provided: convert list → dict and update `params_schema` column
5. Commit; return updated toolkit via existing `get_toolkit` read path

Reuse the existing `FlagDefinition` and `ParamDefinition` models already defined for POST.

### Step 2: UI — fetch helper

**File:** `web_ui/react-src/src/api/client.ts` (or the toolkits API file)

```typescript
export async function patchToolkit(
  id: number,
  payload: BackendToolkitPatchPayload,
): Promise<ToolkitRead> {
  return apiFetch(`/api/toolkits/${id}`, { method: 'PATCH', body: JSON.stringify(payload) })
}
```

Add `BackendToolkitPatchPayload` to `types/index.ts`:

```typescript
export interface BackendToolkitPatchPayload {
  hardware_module_ids?: number[]
  flags?: FlagDefinition[]
  params_schema?: ParamDefinition[]
}
```

### Step 4: UI — EditModal component

**File:** `web_ui/react-src/src/pages/toolkits/Toolkits.tsx` (inline component, same file)

Structure:
- **Header:** "Edit Toolkit: {name}"
- **Three sections** (tabs or stacked — match existing creation modal style):
  1. **HW Modules** — same checkbox list as creation step 3; pre-check existing `hardware_module_ids`
  2. **Flags** — same editable grid as creation step 4; pre-populate from `toolkit.flags`
  3. **Params** — same editable grid as creation step 5; pre-populate from `toolkit.params_schema`
- **Save button:** calls `patchToolkit(toolkit.id, {hardware_module_ids, flags, params_schema})` then invalidates `['toolkits']` cache and closes modal
- **Cancel button:** closes without saving
- Show API error inline if PATCH fails

Pre-population note: `toolkit.flags` arrives from the API as `Record<string, unknown>` (a dict).
Convert to `FlagDefinition[]` for the grid: `Object.entries(flags).map(([name, v]) => ({name, ...v}))`.
Same for `params_schema`.
